# Bash (2:00)

Bash. В чём отличие понятия "шаблон", "маска", "подстрока"?
Подстрока - это конкретная и непрерывная последовательность строковых символов. Она используется для жёсткого сопоставления с реальными строками.
Шаблон и Маска - это взаимозаменяемые понятия. Оба означают некую строку, которая может содержать в себе команыд, выражения, переменные, символы подстановки и регулярные выражения. Они используются для гибкого сопоставления с реальными строками.

Bash. Как найти строки, соответствующие определённому шаблону?
grep -E "<pattern>"

Bash. Как найти строки, которые начинаются с определённой подстроки или шаблона?
grep -E "^<substring|pattern>"

Bash. Как найти строки, которые заканчиваются определённой подстрокой или шаблоном?
grep -E "<substring|pattern>$"

Bash. Как найти и заменить одну подстроку другой подстрокой?
sed 's/<old_substring>/<new_substring>

Bash. Как вывести строки указанного диапазона?
sed -n '<from>,<until>p' <file> # p - print

Bash. Как вывести все строки кроме строк указанного диапазона?
sed '<from>,<until>d' <file> # d - delete

# Pattern matching (2:00)

Pattern matching. Что такое "pattern matching" (сопоставление с шаблоном)?
Проверка соответствия строки (или структуры данных) заданному шаблону (pattern).

Pattern matching. Что такое шаблон (pattern)?
Шаблон (pattern) — это описание формы, структуры или последовательности, с которой мы хотим сопоставить (сравнить) объект: строку, файл, данные, число, структуру и т. д.
Иными словами: шаблон — это правило, по которому мы проверяем совпадение.

Pattern matching. Каких видов бывают шаблоны (patterns)?
Их много, но в рамках написания кода нас интересуют только три:
- Файловые шаблоны — ищут совпадения в метаданных файлов.
- Текстовые шаблоны — ищут совпадения в строках и текстах.
- Структурные шаблоны — распознают и разбирают данные по форме.

Pattern matching. Какие есть системы сопоставления файловых шаблонов?
Только одна: Шаблоны подстановки (Globbing / Wildcard patterns)

Pattern matching. Какие есть системы сопоставления текстовых шаблонов?
Есть три группы систем, на которые ориентируются разработчики при разработке своих инструментов:
- Шаблоны подстановки (Globbing / Wildcard patterns)
- POSIX (BRE - Basic Regular Expressions / ERE - Extended Regular Expressions)
- Perl (PCRE - Perl-Compatible Regular Expressions / PCRE 2)

Pattern matching. Какие есть системы сопоставления структурных шаблонов?
Единых систем нет: у каждого языка она своя.

Pattern matching. Что такое "маска"?
У этого термина две интерпретации:
- Синоним слова "шаблон", но который использует только символы подстановки.
- Элементы языка шаблона (подстановочный символ или конструкция), из которого строится шаблон: *, ?, [a-z], \d, ^, $
Шаблон — это комбинация масок и обычных символов, образующая правило сопоставления.

Pattern matching. Что такое wildcards и globbing?
Wildcards - это синтаксис языка шаблонов.
Globbing - это механизм Unix-подобных оболочек (bash, zsh, fish, etc.), который интерпретирует эти шаблоны и подставляет реальные имена файлов.

# Python (2:15; 1:15)

Python. Как ввести данные с клавиатуры?
input()

Python. Как для ввода с клавиатуры указать подсказку?
input("Hint")

Python. Как в print() изменить разделитель с пробела на другое?
print(sep=<separator>) # В качестве разделителя может выступать строка, так и переменная, но всё равно строка.

Python. Как в print() изменить окончание вывода с "\n" на другое?
print(end=<ending>) # В качестве разделителя может выступать строка, так и переменная, но всё равно строка.

Python. Как называются символы типа "\n"?
Символы управляющие последовательностью.

Python. Что делает управляющий символ "\n"?
Задаёт новую строку.

Python. Что такое множественное присваивание?
Это когда в одном выражении нескольким переменным присваивают значения.
var_1, var_2 = val_1, val_2

Python. В каких случаях применяют множественное присваивание?
Когда надо, чтобы переменные обменялись значениями.
var_1, var_2 = var_2, var_1

Python. Сработает ли такой обмен значениями между переменными?
var_1 = val_1
var_2 = val_2

var_1 = var_2
var_2 = var_1

Нет, потому что python - это интерпретируемый язык, а интерпретатор работает по строчно, а значит в этот момент "var_2 = var_1" у var_1 уже будет значение var_2.
